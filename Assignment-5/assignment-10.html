Q.10 How do you define an asynchronous function in JavaScript using async/await? answer:- To define an asynchronous function in JavaScript using async/await, you simply need to prefix the function declaration with the async keyword. This indicates that the function will perform asynchronous operations and use the await keyword to pause the execution until the promises are resolved or rejected. The await keyword can only be used inside an async function. Here's the basic syntax for defining an asynchronous function with async/await: async function functionName() { // Asynchronous operations using await } Example of an asynchronous function using async/await to fetch data from an API: // Function to simulate API call with a Promise function fetchDataFromServer() { return new Promise((resolve, reject) => { // Simulating an API call delay using setTimeout setTimeout(() => { const data = { id: 1, name: 'John Doe' }; resolve(data); }, 1000); }); } // Asynchronous function using async/await async function getData() { try { // Asynchronous operation using await const data = await fetchDataFromServer(); console.log('Received data:', data); } catch (error) { console.error('Error:', error.message); } } // Calling the asynchronous function getData(); In this example, the getData() function is declared as async, which allows us to use await inside it. Inside the getData() function, we use await to wait for the fetchDataFromServer() Promise to resolve, effectively pausing the execution until the data is available or an error occurs. The data is then logged to the console or, if an error occurs, the error message is caught and logged using the catch block. Using async/await provides a more synchronous-like flow to asynchronous code, making it easier to read, write, and maintain compared to using chained .then() and .catch() with Promises or nested callbacks.
